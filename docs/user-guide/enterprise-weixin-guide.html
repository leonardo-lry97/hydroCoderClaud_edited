企业微信桥接配置指南

开始前先准备好

必要条件
  • 企业已经开通企业微信
  • 你有管理员，或至少有创建和启用智能机器人的权限

第一部分：建立企业微信智能机器人

1

创建主桥接智能机器人

从企业微信桌面端或者 Web 端登录后，先进入工作台，找到“智能机器人”入口。申请入口：企业微信管理后台

先进入工作台，再找到“智能机器人”入口。

进入智能机器人页面后，点击“创建机器人”。

在“创建”页点击“创建机器人”。

在弹出的创建窗口中，先选择“手动创建”。

先走手动创建，不直接用 AI 自动生成。

进入手动创建页后，在页面下方点击“API 模式创建”。

切换到 API 模式后，才能按 Bot ID / Secret 的方式接入。

最后完成主桥接机器人的关键配置：

  • 给机器人改名
  • 修改可见范围
  • 连接方式选择“长连接”
  • 点击获取 Secret，记录 Bot ID 和 Secret
  • 最后保存，注意不要修改可使用权限
主桥接机器人要使用“长连接”，并记录好 Bot ID 和 Secret。
2

创建通讯录辅助机器人

再建立一个辅助机器人，专门用于拉取通讯录，名字可以叫“通讯录机器人”。

创建流程和主桥接智能机器人保持一致，同样是手动创建，再切换到 API 模式创建。

最后完成通讯录辅助机器人的关键配置：

  • 设置机器人名称
  • 修改可见范围
  • 连接方式同样选择“长连接”
  • 记录这个通讯录辅助机器人的 Bot ID 和 Secret
  • 最后一步额外开通“通讯录”权限
通讯录辅助机器人和主桥接机器人创建方式一致，但这里要额外开通“通讯录”权限。

第二部分：Hydro Desktop 配置主桥接机器人

进入设置，渠道配置，选择企业微信。

在页面中开启企业微信开关，并填入如下内容。

字段 怎么填
启用企业微信桥接 打开
Bot ID 填写主桥接智能机器人的 Bot ID
Secret 填写主桥接智能机器人的 Secret
默认工作目录 可选，不填也可以
历史会话数量 可选，默认通常就够用
填写完成后
  • 点击“连接”
  • 等待状态变成“已连接”

第三部分：配置通讯录机器人

在企业微信配置界面中依次执行
  • 检测 CLI
  • 安装 CLI
  • 打开初始化终端
  • 在打开的终端中选择：手动输入 Bot ID 和 Secret
  • 然后依次输入第二步通讯录机器人的 Bot ID 和 Secret
  • 最后显示配置成功
最后执行

测试读取联系人，会弹出读取联系人成功的信息。

第四部分：验证和使用

验证 1：单聊

用企业微信测试账号给机器人发一条普通文本，确认 Hydro Desktop 能收到消息，并且机器人能正常回复。

验证 2：群聊

把机器人加入测试群，在群里发消息，确认桌面端能收到，并且群聊消息不会错落到某个单聊用户会话里。

验证 3：桌面主动发送（可选）

只有在你做了联系人读取初始化后，才验证这一项：在 Hydro Desktop 会话工具栏里点击企业微信发送入口，选择成员发消息，再从企业微信回一条，确认它能回到原会话。

常见问题

配置失败时，先看这些最常见原因。

明明填了 Bot ID 和 Secret，为什么还是不对？

先检查智能机器人是否已经创建成功，消息收发权限是否已经开通，以及配置是否已经生效。

为什么桌面端主动发送列表为空？

通常先检查 wecom-cli 是否已安装、是否完成初始化、联系人读取授权是否完成，以及是否把桥接聊天和联系人读取混在一起用了。

为什么企业微信和飞书 / 钉钉表现不太一样？

因为企业微信当前保留了原生流式回复能力，所以你看到的回复节奏可能和飞书、钉钉不同，这是正常现象。

为什么建议不要复用桥接机器人的 Bot ID / Secret 做联系人读取？

因为桥接聊天和联系人读取是两条不同链路。拆开配置更稳妥，也更不容易在排障时混淆问题来源。
