# 飞书桥接使用指南

通过飞书机器人桥接，你可以在飞书中直接与 Hydro Desktop 的 Agent 对话，实现远程访问本地 Agent 会话。

## 前提条件

- Hydro Desktop 已安装并正常运行
- Claude Code CLI 已安装且 API 配置正常
- 拥有飞书组织管理员权限

## 重要限制

> 一个飞书应用只能绑定一个桌面端。多台桌面端同时连接同一个应用，可能导致消息丢失或重复回复。

## 配置步骤

### 1. 创建飞书企业自建应用

1. 登录 [飞书开放平台](https://open.feishu.cn/)
2. 进入「开发者后台」→「创建企业自建应用」
3. 创建后记录 `App ID` 与 `App Secret`

### 2. 开通机器人能力与权限

至少需要以下能力：

- 获取用户信息
- 获取通讯录信息
- 发送消息
- 上传图片
- 读取图片资源

说明：

- 飞书桥接使用 WebSocket 长连接接收入站，不需要配置 HTTP 回调 URL
- 如权限有变更，需重新发布应用

### 3. 在 Hydro Desktop 中配置

填写以下内容：

| 字段 | 说明 |
|------|------|
| 启用飞书桥接 | 打开开关 |
| App ID | 飞书应用凭据 |
| App Secret | 飞书应用凭据 |
| 默认工作目录 | Agent 默认 cwd，可选 |
| 历史会话数量 | `/status` `/resume` 等菜单展示数量 |

保存后点击「连接」，状态显示为「已连接」即可。

## 使用方式

### 基本对话

在飞书单聊或群聊中直接发送文本消息即可。

当前行为：

- 单聊按用户维度维护会话
- 群聊按群维度维护会话
- 同一群中的不同成员，不会各自拆成不同桌面会话

### 图片消息

- 飞书发送图片到机器人，桌面端会把图片作为会话输入交给 Agent
- Agent 在处理过程中读取的本地图片，也会回传到飞书

### 桌面主动发飞书

在桌面端聊天输入框工具栏点击飞书图标，可选择联系人或群聊发送消息。

当前行为：

1. 首次主动发送成功后，会把当前桌面会话绑定到该飞书目标
2. 后续该目标从飞书回复时，会优先回到当前绑定会话
3. 如果当前会话已经关闭，则进入历史选择 / 新建逻辑

## 命令系统

在飞书中发送以 `/` 开头的消息，会走命令链路，不进入 AI 对话。

### 当前可用命令

| 命令 | 说明 |
|------|------|
| `/help` | 显示帮助 |
| `/status` | 查看当前聊天上下文下的历史会话状态 |
| `/sessions` | 列出活跃会话 |
| `/new [目录]` | 新建会话 |
| `/resume [编号]` | 恢复历史会话 |
| `/rename <名称>` | 重命名当前会话 |
| `/close` | 关闭当前会话 |

### 当前命令展示方式

飞书本轮已经移除交互卡片主方案，当前命令结果统一走文本 / Markdown 风格回复，不再依赖卡片按钮。

### `/status` 的状态标记

`/status` 中历史会话列表的标记语义为：

- `✅` 当前映射会话
- `🔵` 已激活但不是当前映射
- `⭕` 历史存在但当前未激活

这里的 `✅` 表示“当前聊天上下文正在使用的会话”，不是简单的“会话已激活”。

### `/resume` 的行为

1. 不带编号时，显示历史列表
2. 带编号时，直接恢复指定会话
3. 若目标会话已经在运行且当前正在回复，则提示“当前正在回复，请等待完成”
4. 若目标会话未激活，则先恢复，再在需要时自动触发 hello 激活

### `/close` 的行为

- 当前只支持直接使用 `/close`
- 不支持 `/close 1` 这类带编号形式

## 会话管理

### 关闭后再次入站

如果你关闭了当前桌面会话，再从同一飞书聊天发送消息：

1. 如果仍存在当前可复用绑定，会回到该绑定会话
2. 否则会进入历史选择 / 新建逻辑

不会再单纯依赖旧 `type` 或旧字段进行误判。

### 群聊 mention 处理

飞书群聊中：

- `@机器人` 不应污染正文
- 其他用户的 mention 也会尽量从送给模型的正文中剥离，减少干扰

## 桌面端介入

飞书发起的会话在桌面端可见。桌面端用户在该会话中继续发言时：

1. Agent 回复会回传到当前绑定的飞书目标
2. 不会误回到其他 IM 渠道或其他飞书聊天

## 常见问题

### 连接失败

- 检查 `App ID` / `App Secret`
- 检查应用是否已发布
- 检查机器人权限是否已生效

### 消息无回复

- 检查 Hydro Desktop 是否显示「已连接」
- 检查 Claude Code CLI 的 API 配置
- 查看主进程日志或开发者工具日志

### 桌面端看不到可发送成员

- 检查通讯录权限
- 权限变更后重新发布应用

### 收到重复回复

- 检查是否有多台桌面端共用同一个飞书应用
