# 飞书桥接配置指南

这份文档只解决一件事：**让用户按步骤把飞书和 Hydro Desktop 配通**。

## 先准备好这些条件

开始前请先确认：

- 你所在的企业已经开通飞书
- 你有 **企业管理员**、**应用管理员**，或至少有权限创建和发布企业自建应用的账号
- 电脑上已经安装并能正常打开 Hydro Desktop
- Hydro Desktop 里的 Agent / 模型能力已经可用，能够正常发起本地对话
- 这台电脑在你使用期间会持续在线；不要让多台桌面端同时连接同一个飞书应用

建议提前准备：

- 一个用于测试的飞书账号
- 一个飞书群聊，用来验证群聊场景

## 第 1 步：在飞书开放平台创建应用

1. 登录飞书开放平台
   - 申请入口：https://open.feishu.cn/
2. 进入开发者后台
3. 创建 **企业自建应用**
4. 给应用起一个容易识别的名字，例如“Hydro Desktop 助手”
5. 创建完成后，进入应用详情页，记录：
   - `App ID`
   - `App Secret`

## 第 2 步：给应用开启机器人能力

在应用后台中：

1. 找到 **应用能力**
2. 开启 **机器人** 能力

说明：

- Hydro Desktop 的飞书桥接是通过“应用机器人”收发消息的
- 如果没有开启机器人能力，后面即使填了 `App ID` 和 `App Secret` 也无法正常聊天

## 第 3 步：配置权限和事件订阅

### 需要开的能力

为了保证能正常聊天、发图片、读取联系人，建议至少开通这些方向的权限：

- 消息接收
- 消息发送
- 图片上传 / 图片读取
- 用户信息读取
- 通讯录 / 部门成员读取
- 获取群组信息

如果你不确定具体要开哪些权限，可以全选开通消息与群组、通讯录的全部权限。

### 事件订阅怎么配

1. 进入应用后台的 **事件订阅**
2. 选择 **长连接接收事件**
3. 订阅消息接收事件

当前最关键的是：

- `im.message.receive_v1`

说明：

- Hydro Desktop 当前使用的是 **长连接模式**
- 不需要你额外准备公网回调地址

## 第 4 步：发布应用

前面的能力、权限、事件订阅配完后：

1. 在飞书开放平台提交应用发布
2. 等待发布生效

注意：

- 如果你后面又改了权限或事件订阅，通常还要重新发布一次

## 第 5 步：在 Hydro Desktop 中填写配置

打开 Hydro Desktop，进入 **飞书桥接设置**，按下面填写：

| 字段 | 填什么 |
|------|------|
| 启用飞书桥接 | 打开 |
| App ID | 飞书开放平台里的 `App ID` |
| App Secret | 飞书开放平台里的 `App Secret` |
| 默认工作目录 | 可选，不填也可以 |
| 历史会话数量 | 可选，保持默认通常就够用 |

填写完成后：

1. 点击 **保存**
2. 点击 **连接**
3. 等待状态变成 **已连接**

## 第 6 步：验证是否配置成功

建议按下面顺序验证：

### 验证 1：单聊

1. 用飞书测试账号给机器人发一条普通文字消息
2. 看 Hydro Desktop 是否收到消息并创建 / 命中会话
3. 看机器人是否能正常回复

### 验证 2：桌面主动发送

1. 在 Hydro Desktop 会话工具栏里点击飞书发送入口
2. 选择联系人
3. 发送一条测试消息
4. 再从飞书回复一条消息，确认它能回到原会话

### 验证 3：群聊

1. 把机器人加入测试群
2. 在群里发送消息
3. 确认桌面端能收到，并且群聊消息不会错落到某个单聊用户会话里

## 常见问题

### 明明填了 App ID 和 App Secret，为什么还是连不上

优先检查：

- 应用是否已经发布生效
- 机器人能力是否已开启
- 事件订阅是否配置为长连接
- 消息接收事件是否已订阅

### 为什么桌面端看不到可发送联系人

通常是通讯录 / 部门成员读取权限不够，或者权限改完后还没有重新发布应用。

### 为什么飞书里发消息没有反应

优先检查：

- Hydro Desktop 里飞书桥接状态是不是“已连接”
- 当前电脑上的 Agent / 模型功能是否本来就可用
- 是否有多台桌面端同时连接了同一个飞书应用

### 为什么我建议只让一台桌面端连接这个应用

因为多台桌面端同时接同一个飞书应用时，最容易出现消息重复、抢消息或回复错乱。
