# 飞书桥接使用指南

通过飞书机器人桥接，你可以在飞书中直接与 Hydro Desktop 的 Agent 对话，实现远程访问本地 Agent 会话。

## 前提条件

- Hydro Desktop 已安装并正常运行
- Claude Code CLI 已安装且 API 配置正常
- 拥有飞书组织的管理员权限（用于创建企业自建应用）

## 重要限制

> **一个飞书应用只能绑定一个桌面端。** 同一组织内，同一个应用的 App ID/App Secret 只能在一台 Hydro Desktop 上配置使用。多台桌面端同时连接同一个应用会导致消息丢失或重复回复。如需多人使用，请为每人创建独立的企业自建应用。

## 第一步：创建飞书企业自建应用

1. 登录 [飞书开放平台](https://open.feishu.cn/)
2. 进入「开发者后台」→「创建企业自建应用」
3. 填写应用名称（如 "Hydro Desktop AI"）和描述
4. 创建完成后，在应用详情页「凭证与基础信息」中获取 **App ID** 和 **App Secret**

## 第二步：配置机器人能力

1. 在应用详情页，进入「添加应用能力」→ 添加「机器人」
2. 配置机器人信息：
   - 机器人名称：自定义（如 "Hydro Desktop Agent"）
3. 在「权限管理」中确保已开通以下权限：
   - **获取用户信息**（用于解析消息发送者名称）
   - **获取通讯录信息**（用于桌面端主动发送时选择联系人）
   - **发送消息**（用于回复消息与主动推送）
   - **上传图片**（用于图片转发）
   - **读取图片资源**（用于接收飞书端发送的图片）
4. 在「安全设置」中配置 IP 白名单（如需要）
5. 发布应用并等待管理员审批通过（未发布的应用无法使用）

### 事件订阅

飞书桥接使用 WebSocket 长连接方式接收消息，此方式**无需配置 HTTP 请求 URL**。

如果之前已配置了事件订阅 URL，请确保：
- 不删除机器人相关的事件订阅
- 不修改与 `im.message.receive_v1` 相关的事件配置

## 第三步：在 Hydro Desktop 中配置

1. 打开 Hydro Desktop
2. 点击左下角齿轮图标，选择「飞书设置」（或在设置页切换到飞书 tab）
3. 填写配置：

   | 字段 | 说明 |
   |------|------|
   | **启用飞书桥接** | 打开开关 |
   | **App ID** | 填入第一步获取的 App ID |
   | **App Secret** | 填入第一步获取的 App Secret |
   | **默认工作目录** | Agent 执行命令的工作目录（可选，强烈建议设置） |
   | **历史会话数量** | 飞书用户选择历史会话时显示的最大数量（默认 5，范围 1-20） |

4. 点击「保存」，然后点击「连接」
5. 状态显示「已连接」即配置成功

## 使用方式

### 基本对话

在飞书中找到机器人，直接发送文本消息即可。机器人会将消息转发给 Agent，并将回复发回飞书。

### 发送图片给 Agent

在飞书对话中直接发送图片，Agent 会自动识别图片内容并回复。

### Agent 图片转发

当 Agent 在处理过程中使用工具读取了本地图片文件（如通过 Read 工具），该图片会自动转发到飞书。

**提示**：如果希望 Agent 生成图片后自动发送到飞书，可以在提示词中加上"生成后请读取该图片"，确保 Agent 使用 Read 工具读取图片文件。

### 桌面主动发飞书

在桌面端聊天输入框工具栏点击飞书图标，可直接选择组织成员或群聊发送消息。

当前行为是：

- 首次从某个桌面会话主动发给飞书联系人后，该联系人会绑定到当前桌面会话
- 后续该联系人从飞书回复时，会优先回到当前桌面会话
- 如果当前会话已关闭，再回复时才会回到历史/新建逻辑

## 命令系统

在飞书中发送以 `/` 开头的消息，会被识别为命令，不进入 AI 对话。飞书使用交互式卡片展示命令结果。

### 命令列表

| 命令 | 说明 |
|------|------|
| `/help` | 显示所有可用命令（卡片形式） |
| `/status` | 查看系统状态（连接状态、当前会话、模型、执行中/空闲数） |
| `/sessions` | 列出当前聊天下的活跃会话 |
| `/new [目录]` | 新建会话，可选指定工作目录名或绝对路径 |
| `/resume [编号]` | 恢复历史会话，不带编号时显示选择菜单 |
| `/rename <名称>` | 修改当前会话名称 |
| `/close [编号]` | 关闭当前会话或指定会话（编号以 `/sessions` 列表为准） |

### 使用示例

**新建会话并指定工作目录**

```
/new my-project
```

会在默认输出目录下创建 `my-project` 子目录作为工作目录。

```
/new /Users/yourname/projects/webapp
```

macOS 使用绝对路径直接指定工作目录。

```
/new D:\projects\webapp
```

Windows 使用绝对路径直接指定工作目录。

**恢复历史会话**

```
/resume
```

显示历史会话列表，点击卡片按钮即可恢复对应会话。

```
/resume 2
```

直接恢复列表中第 2 个历史会话。

### 注意事项

- 飞书使用交互卡片展示命令菜单，点击卡片按钮选择操作
- 卡片发送失败时会自动回退到纯文本菜单
- `/resume` 仅在当前会话正在处理中（streaming）时不可用；空闲会话下可直接恢复历史会话
- `/resume` 切换的是当前飞书对话绑定的会话，不会自动关闭其他空闲会话
- `/rename` 和 `/close` 需要有活跃会话才能执行

## 会话管理

### 会话隔离

不同单聊或群聊发起的对话相互隔离，各自维护独立的 Agent 会话和上下文。同一个群里的消息共用一个会话。

### 历史会话选择

当当前飞书对话没有绑定活跃会话、但存在历史会话时，机器人会列出历史会话供选择（卡片或文本菜单形式）。

- 选择**新建会话**：创建全新会话，从头开始
- 选择**历史会话**：恢复对应历史会话（包含历史上下文，Agent 可继续之前的对话）

### 正在处理时发消息

如果 Hydro Desktop 桌面端正在处理某个请求（Agent 生成回复中），此时飞书发来的消息会收到提示：

```
AI 正在响应中，请等待完成后再发送下一条消息
```

等 Agent 回复完成后再发送即可。

## Hydro Desktop 桌面端介入

### 什么是介入

飞书发起的会话同时在 Hydro Desktop 桌面端可见。当桌面端用户（而非飞书用户）在该会话窗口中直接输入问题并获得 Agent 回复时，这次交互称为"桌面端介入"。

### 介入内容自动同步

桌面端介入的完整 Q&A 会自动发送到飞书，格式如下：

```
💻 桌面端介入：
> 帮我检查一下最新的日志文件

日志中发现 3 处异常，分别位于...（Agent 完整回复）
```

如果 Agent 在介入过程中读取了图片文件，图片同样会转发到飞书。

## 常见问题

### 连接失败

- 确认 App ID 和 App Secret 是否正确
- 确认应用已发布并通过审批
- 确认应用已添加机器人能力
- 检查网络是否能访问飞书服务器

### 消息无回复

- 确认 Hydro Desktop 状态栏显示「已连接」
- 确认 Claude Code CLI 的 API 配置正常
- 查看 Hydro Desktop 开发者工具（F12）中的控制台日志

### 图片无法转发到飞书

- 确认已开通「上传图片」权限
- 确认图片文件大小不超过 20MB
- 确认 Agent 确实使用了 Read 工具读取图片（仅文本中提到路径不会触发转发）

### 收到重复回复

- 检查是否有多台桌面端配置了同一个飞书应用，请确保一个应用只绑定一个桌面端

### 为什么桌面端看不到可发送成员

- 确认已开通「获取通讯录信息」权限
- 确认应用已发布
- 权限变更后需重新发布应用
