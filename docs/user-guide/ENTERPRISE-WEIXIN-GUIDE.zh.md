# 企业微信桥接配置指南

这份文档只解决一件事：**让用户按步骤把企业微信和 Hydro Desktop 配通**。

## 先准备好这些条件

开始前请先确认：

- 你所在的企业已经开通企业微信
- 你有 **企业微信管理员**，或至少有权限创建和发布智能机器人的账号
- 电脑上已经安装并能正常打开 Hydro Desktop
- Hydro Desktop 里的 Agent / 模型功能已经可正常使用
- 不要让多台桌面端同时连接同一个企业微信机器人

如果你还需要从桌面端主动给企业微信成员发消息，请额外确认：

- 你愿意额外完成 `wecom-cli` 初始化
- 你可以准备一个 **单独用于联系人读取** 的机器人或授权能力

## 第 1 步：在企业微信创建桥接机器人

1. 登录企业微信管理后台
   - 申请入口：https://work.weixin.qq.com/
2. 创建 **智能机器人**
3. 创建完成后，记录：
   - `Bot ID`
   - `Secret`

说明：

- 这一步创建的是桥接主机器人
- 它负责接收企业微信消息，并把消息带回 Hydro Desktop

## 第 2 步：给桥接机器人开通权限

桥接主链路至少需要这些能力：

- 消息收发
- 媒体文件上传

说明：

- 如果这些权限没开好，文本和图片链路都可能不完整

## 第 3 步：发布或启用机器人

前面的配置完成后：

1. 确认机器人配置已经生效
2. 如果后台有发布 / 生效动作，按企业微信当前控制台要求完成

## 第 4 步：在 Hydro Desktop 中填写桥接配置

打开 Hydro Desktop，进入 **企业微信桥接设置**，按下面填写：

| 字段 | 填什么 |
|------|------|
| 启用企业微信桥接 | 打开 |
| Bot ID | 企业微信后台里的 `Bot ID` |
| Secret | 企业微信后台里的 `Secret` |
| 默认工作目录 | 可选，不填也可以 |
| 历史会话数量 | 可选，默认通常就够用 |

填写完成后：

1. 点击 **保存**
2. 点击 **连接**
3. 等待状态变成 **已连接**

## 第 5 步：如果需要“主动给成员发消息”，再做这一步

这一部分是 **可选项**。

如果你只是想让企业微信用户发消息给机器人，然后在桌面端继续处理，那么前面的步骤已经够了。

只有在你需要“从桌面端主动选择企业微信成员发消息”时，才需要继续做：

1. 安装并初始化 `wecom-cli`
2. 完成联系人读取相关授权
3. 建议使用 **独立的联系人读取机器人 / 配置**
4. 不要复用桥接机器人的 `Bot ID / Secret`

这样做的原因是：

- 桥接机器人负责聊天主链路
- 联系人读取是另一条能力链路
- 把两者拆开更稳妥，也更容易排障

## 第 6 步：验证是否配置成功

建议按下面顺序验证：

### 验证 1：单聊

1. 用企业微信测试账号给机器人发一条普通文字消息
2. 看 Hydro Desktop 是否收到消息并创建 / 命中会话
3. 看机器人是否能正常回复

### 验证 2：群聊

1. 把机器人加入测试群
2. 在群里发送消息
3. 确认桌面端能收到，并且群聊消息不会错落到某个单聊用户会话里

### 验证 3：桌面主动发送（可选）

只有你做了第 5 步时才验证这一项：

1. 在 Hydro Desktop 会话工具栏里点击企业微信发送入口
2. 选择成员发送一条测试消息
3. 再从企业微信回一条消息，确认它能回到原会话

## 常见问题

### 明明填了 Bot ID 和 Secret，为什么还是不对

优先检查：

- 智能机器人是否已经创建成功
- 消息收发权限是否已经开通
- 配置是否已经生效

### 为什么桌面端主动发送列表为空

通常先检查：

- `wecom-cli` 是否已经安装
- 是否已经完成初始化
- 联系人读取相关授权是否完成
- 是否把桥接机器人和联系人读取能力混在一起用了

### 为什么企业微信和飞书 / 钉钉表现不太一样

因为企业微信当前保留了原生流式回复能力，所以你看到的回复节奏可能和飞书、钉钉不同，这是正常现象。

### 为什么建议不要复用桥接机器人的 Bot ID / Secret 做联系人读取

因为桥接聊天和联系人读取是两条不同链路。拆开配置更稳妥，也更不容易在排障时混淆问题来源。
