# 企业微信桥接使用指南

通过企业微信智能机器人桥接，你可以在企业微信中直接与 Hydro Desktop 的 Agent 对话，支持流式回复和桌面主动推送。

## 前提条件

- Hydro Desktop 已安装并正常运行
- Claude Code CLI 已安装且 API 配置正常
- 拥有企业微信管理后台的管理员权限（用于创建智能机器人）

## 重要限制

> **一个智能机器人只能绑定一个桌面端。** 同一企业内，同一个机器人的 Bot ID/Secret 只能在一台 Hydro Desktop 上配置使用。多台桌面端同时连接同一个机器人会导致消息丢失或重复回复。如需多人使用，请为每人创建独立的智能机器人。

## 第一步：创建企业微信智能机器人

1. 登录 [企业微信管理后台](https://work.weixin.qq.com/)
2. 进入「应用管理」→「智能机器人」→「创建智能机器人」
3. 填写机器人名称（如 "Hydro Desktop AI"）和描述
4. 创建完成后，在机器人详情页获取 **Bot ID** 和 **Secret**

> **注意**：这里的 Bot ID 和 Secret 仅用于消息收发桥接，不要用于后文提到的 wecom-cli 联系人读取。

## 第二步：配置机器人权限

在机器人详情页或「应用管理」中确保已开通以下权限：

- **消息收发**（基础权限，创建机器人即默认开通）
- **媒体文件上传**（用于图片转发到企业微信）

如需在企业微信端自动弹出联系人授权确认，可在机器人设置中开启「进入会话时发送欢迎语」。

## 第三步：在 Hydro Desktop 中配置

### 基本配置

1. 打开 Hydro Desktop
2. 点击左下角齿轮图标，选择「企业微信设置」（或在设置页切换到企业微信 tab）
3. 填写基本配置：

   | 字段 | 说明 |
   |------|------|
   | **启用企业微信桥接** | 打开开关 |
   | **Bot ID** | 填入第一步获取的 Bot ID |
   | **Secret** | 填入第一步获取的 Secret |
   | **默认工作目录** | Agent 执行命令的工作目录（可选，强烈建议设置） |
   | **历史会话数量** | 企业微信用户选择历史会话时显示的最大数量（默认 5，范围 1-20） |

4. 点击「保存」，然后点击「连接」
5. 状态显示「已连接」即桥接配置成功

### 联系人获取（wecom-cli 集成）

桌面端向企业微信成员主动发送消息需要读取组织通讯录。企业微信的联系人通过 `wecom-cli` CLI 工具按需读取。

> **重要安全提醒**：联系人读取请务必使用**新建的独立机器人**完成初始化，**不要复用**当前桥接机器人的 Bot ID/Secret，也不要给桥接机器人开通通讯录权限。

配置步骤：

1. **安装 CLI**：点击「安装 CLI」按钮，系统会打开终端执行安装命令
2. **初始化**：点击「打开初始化终端」按钮，系统打开可见终端并启动 `wecom-cli init`
   - 在终端内按提示完成接入方式选择
   - 输入**新建的独立机器人**的 Bot ID/Secret（不是桥接机器人的凭据）
   - 如果企业微信端没有弹出联系人授权确认，请改为前往企业微信管理后台手动为**该独立机器人**开通通讯录权限
3. **测试联系人**：点击「测试读取联系人」按钮确认通讯录权限生效

CLI 检测界面显示三个状态指示灯：

| 状态 | 说明 |
|------|------|
| CLI 安装 | 显示 `wecom-cli` 工具是否已安装 |
| CLI 初始化 | 显示 `.config/wecom/bot.enc` 配置是否就绪 |
| 机器人通讯录授权 | 显示独立机器人的通讯录访问权限状态 |

## 使用方式

### 基本对话

在企业微信中找到机器人（单聊或群聊），直接发送文本消息即可。机器人会将消息转发给 Agent，并将回复以流式方式发回企业微信。

企业微信的回复是**原生流式输出**——Agent 生成的内容会实时逐段发送，不需要等待完整回复。

### 发送图片给 Agent

在企业微信对话中直接发送图片（支持单张图片和图文混排），Agent 会自动识别图片内容并回复。

### Agent 图片转发

当 Agent 在处理过程中使用工具读取了本地图片文件（如通过 Read 工具），该图片会自动转发到企业微信。

**提示**：如果希望 Agent 生成图片后自动发送到企业微信，可以在提示词中加上"生成后请读取该图片"，确保 Agent 使用 Read 工具读取图片文件。

### 桌面主动发企业微信

在桌面端聊天输入框工具栏点击企业微信图标，可直接选择组织成员发送消息。

当前行为是：

- 首次从某个桌面会话主动发给企业微信联系人后，该联系人会绑定到当前桌面会话
- 后续该联系人从企业微信回复时，会优先回到当前桌面会话
- 如果当前会话已关闭，再回复时才会回到历史/新建逻辑

> **注意**：桌面端主动发送功能依赖 wecom-cli 联系人获取（见第三步）。如果未完成 CLI 初始化和通讯录授权，可发送成员列表将为空。

## 命令系统

在企业微信中发送以 `/` 开头的消息，会被识别为命令，不进入 AI 对话。

### 命令列表

| 命令 | 说明 |
|------|------|
| `/help` | 显示所有可用命令 |
| `/status` | 查看当前会话状态（含历史会话列表） |
| `/sessions` | 列出当前聊天下的活跃会话 |
| `/new [目录]` | 新建会话，可选指定工作目录名或绝对路径 |
| `/resume [编号]` | 恢复历史会话，不带编号时显示选择列表 |
| `/rename <名称>` | 修改当前会话名称 |
| `/close [编号]` | 关闭当前会话或指定会话（编号以 `/sessions` 列表为准） |

### 使用示例

**新建会话并指定工作目录**

```
/new my-project
```

会在默认输出目录下创建 `my-project` 子目录作为工作目录。

```
/new /Users/yourname/projects/webapp
```

macOS 使用绝对路径直接指定工作目录。

```
/new D:\projects\webapp
```

Windows 使用绝对路径直接指定工作目录。

**恢复历史会话**

```
/resume
```

显示历史会话列表，回复对应编号即可恢复。

```
/resume 2
```

直接恢复列表中第 2 个历史会话。

**重命名当前会话**

```
/rename 前端重构任务
```

**查看系统状态**

```
/status
```

返回当前会话下的历史会话列表。

### 注意事项

- 命令不区分大小写，`/Help` 和 `/help` 效果相同
- `/resume` 仅在当前会话正在处理中（streaming）时不可用
- `/resume` 切换的是当前企业微信对话绑定的会话，不会自动关闭其他空闲会话
- `/rename` 和 `/close` 需要有活跃会话才能执行

## 会话管理

### 会话隔离

不同单聊或群聊发起的对话相互隔离，各自维护独立的 Agent 会话和上下文。同一个群里的消息共用一个会话。

### 历史会话选择

当当前企业微信对话没有绑定活跃会话、但存在历史会话时，机器人会列出历史会话供选择：

```
您有以下历史会话，请回复数字选择：

1. ✅ [2小时前] 代码审查任务 (my-project) 官方 API
2. ⭕ [昨天] Python 脚本生成 (scripts) 官方 API

回复 0 开始全新会话
```

- 回复 **0**：创建全新会话，从头开始
- 回复 **1~N**：恢复对应历史会话（包含历史上下文，Agent 可继续之前的对话）

**等待选择期间继续发消息**：如果你在历史会话选择期间发送 `/new`，会直接新建会话；发送其他文本会被视为无效输入，机器人会重新显示选择菜单。

### 正在处理时发消息

如果 Hydro Desktop 桌面端正在处理某个请求（Agent 生成回复中），此时企业微信发来的消息会收到提示：

```
AI 正在响应中，请等待完成后再发送下一条消息
```

等 Agent 回复完成后再发送即可。

## Hydro Desktop 桌面端介入

### 什么是介入

企业微信发起的会话同时在 Hydro Desktop 桌面端可见。当桌面端用户（而非企业微信用户）在该会话窗口中直接输入问题并获得 Agent 回复时，这次交互称为"桌面端介入"。

### 介入内容自动同步

桌面端介入的完整 Q&A 会自动发送到企业微信，格式如下：

```
💻 桌面端介入：
> 帮我检查一下最新的日志文件

日志中发现 3 处异常，分别位于...（Agent 完整回复）
```

如果 Agent 在介入过程中读取了图片文件，图片同样会转发到企业微信。

## 常见问题

### 连接失败

- 确认 Bot ID 和 Secret 是否正确（注意不是 wecom-cli 机器人的凭据）
- 确认智能机器人已创建并启用
- 检查网络是否能访问企业微信服务器

### 消息无回复

- 确认 Hydro Desktop 状态栏显示「已连接」
- 确认 Claude Code CLI 的 API 配置正常
- 查看 Hydro Desktop 开发者工具（F12）中的控制台日志

### 图片无法转发到企业微信

- 确认已开通「媒体文件上传」权限
- 确认图片文件大小不超过 20MB
- 确认 Agent 确实使用了 Read 工具读取图片（仅文本中提到路径不会触发转发）

### 桌面端主动发送列表为空

- 确认已完成 wecom-cli 安装和初始化
- 确认独立机器人的通讯录权限已授权
- 点击「测试读取联系人」确认权限状态
- 如果提示未授权，去企业微信管理后台为独立机器人手动开通通讯录权限

### 收到重复回复

- 检查是否有多台桌面端配置了同一个智能机器人，请确保一个机器人只绑定一个桌面端

### 桥接机器人和联系人读取机器人的区别

| | 桥接机器人 | 联系人读取机器人 |
|------|---------|---------|
| 用途 | 收发消息 | 读取通讯录 |
| 凭据填入位置 | 基本配置 (Bot ID/Secret) | wecom-cli init (终端输入) |
| 权限要求 | 消息收发、媒体上传 | 通讯录读取 |
| 是否可复用 | 否（必须独立） | 否（必须独立） |
| 桌面端配置 | 直接输入 | 通过 wecom-cli 管理 |

两个机器人必须独立创建，各司其职，不能互换或复用凭据。
