# 企业微信桥接使用指南

通过企业微信智能机器人桥接，你可以在企业微信中直接与 Hydro Desktop 的 Agent 对话，支持原生流式回复和桌面主动推送。

## 前提条件

- Hydro Desktop 已安装并正常运行
- Claude Code CLI 已安装且 API 配置正常
- 拥有企业微信管理员权限

## 重要限制

> 一个智能机器人只能绑定一个桌面端。多台桌面端同时连接同一个机器人，可能导致消息丢失或重复回复。

## 配置步骤

### 1. 创建桥接机器人

1. 登录 [企业微信管理后台](https://work.weixin.qq.com/)
2. 创建智能机器人
3. 记录 `Bot ID` 与 `Secret`

### 2. 开通机器人权限

桥接主链路至少需要：

- 消息收发
- 媒体文件上传

### 3. 在 Hydro Desktop 中配置

| 字段 | 说明 |
|------|------|
| 启用企业微信桥接 | 打开开关 |
| Bot ID | 桥接机器人凭据 |
| Secret | 桥接机器人凭据 |
| 默认工作目录 | Agent 默认 cwd，可选 |
| 历史会话数量 | 历史会话菜单数量 |

保存后点击「连接」，状态显示「已连接」即可。

### 4. 配置联系人读取（可选）

桌面端主动给企业微信成员发消息，依赖 `wecom-cli` 联系人读取能力。

重要约束：

- 联系人读取建议使用独立机器人完成初始化
- 不要复用桥接机器人的 `Bot ID / Secret`

如果不需要桌面主动给成员发消息，可以跳过这一部分。

## 使用方式

### 基本对话

在企业微信单聊或群聊中直接发送文本消息即可。

当前行为：

- 单聊按用户维度管理会话
- 群聊按群维度管理会话
- 回复会以企业微信原生流式方式逐段返回

### 图片消息

- 企业微信图片可进入桌面会话
- Agent 读取的本地图片可回传企业微信

### 桌面主动发企业微信

在桌面端聊天输入框工具栏点击企业微信图标，可选择成员发送；对于已识别群聊，可按当前实现作为目标复用。

当前行为：

1. 首次主动发送成功后，当前桌面会话绑定到该企业微信目标
2. 后续该目标从企业微信回复时，会优先回到当前绑定会话
3. 若当前会话已关闭，则进入历史选择 / 新建逻辑

## 命令系统

企业微信发送以 `/` 开头的消息，会走命令链路。

### 当前可用命令

| 命令 | 说明 |
|------|------|
| `/help` | 显示帮助 |
| `/status` | 查看当前聊天上下文下的历史会话状态 |
| `/sessions` | 列出活跃会话 |
| `/new [目录]` | 新建会话 |
| `/resume [编号]` | 恢复历史会话 |
| `/rename <名称>` | 重命名当前会话 |
| `/close` | 关闭当前会话 |

### `/status` 的状态标记

- `✅` 当前映射会话
- `🔵` 已激活但不是当前映射
- `⭕` 历史存在但当前未激活

### `/resume` 的行为

1. 不带编号时，显示历史列表
2. 带编号时，直接恢复目标会话
3. 若目标会话正在回复，则提示“当前正在回复，请等待完成”
4. 若目标会话未激活，则恢复后按需要自动 hello 激活

### `/close` 的行为

- 当前只支持 `/close`
- 不支持 `/close 1` 这类带编号形式

## 会话管理

### 群聊名称

企业微信群名通常无法始终从平台稳定获得，当前界面展示会优先使用：

1. 本地 alias / 已知群名
2. 最近识别到的群名称
3. 最后回退到 chatId

因此你看到的群名称，可能是本地别名，而不是平台即时返回值。

### 关闭后再次入站

当前桌面会话关闭后，再从同一企业微信聊天发消息：

1. 如果仍存在允许复用的当前绑定，则回到该绑定会话
2. 否则进入历史选择 / 新建逻辑

## 桌面端介入

企业微信发起的会话在桌面端可见。桌面端继续发言时：

1. 回复会回传到当前绑定企业微信目标
2. 不会误回到其他渠道

## 常见问题

### 桌面端主动发送列表为空

优先检查：

- `wecom-cli` 是否已安装
- 是否完成初始化
- 联系人读取机器人是否已授权通讯录

### 为何企业微信是流式回复

因为当前实现保留了企业微信原生 `replyStream` 能力，这也是它与飞书、钉钉的主要平台差异之一。

### 收到重复回复

检查是否多台桌面端共用同一个企业微信机器人。
