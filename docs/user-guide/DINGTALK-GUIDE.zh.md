# 钉钉机器人桥接使用指南

通过钉钉机器人桥接，你可以在钉钉中直接与 Hydro Desktop 的 Agent 对话，实现远程访问本地 Agent 会话。

## 前提条件

- Hydro Desktop 已安装并正常运行
- Claude Code CLI 已安装且 API 配置正常
- 拥有钉钉组织管理员权限

## 重要限制

> 一个钉钉机器人只能绑定一个桌面端。多台桌面端同时连接同一个机器人，可能导致消息丢失或重复回复。

## 配置步骤

### 1. 创建钉钉机器人应用

1. 登录 [钉钉开放平台](https://open-dev.dingtalk.com/)
2. 创建企业内部开发应用
3. 添加机器人能力
4. 记录 `AppKey`、`AppSecret`、`robotCode`

### 2. 开通最小权限

如果需要桌面端主动发送，至少需要：

- 上传媒体文件
- 机器人发消息
- `qyapi_get_department_list`
- `qyapi_get_department_member`

修改权限后需要重新发布应用。

### 3. 在 Hydro Desktop 中配置

| 字段 | 说明 |
|------|------|
| 启用钉钉桥接 | 打开开关 |
| AppKey | 应用凭据 |
| AppSecret | 应用凭据 |
| robotCode | 机器人编码，主动发送必填 |
| 默认工作目录 | Agent 默认 cwd，可选 |
| 历史会话数量 | 历史会话菜单数量 |

保存后点击「连接」，状态显示「已连接」即可。

## 使用方式

### 基本对话

在钉钉单聊或群聊中直接发送文本消息即可。

当前行为：

- 单聊按用户维度管理会话
- 群聊按群维度管理会话
- 群聊不会再错误落到某个单聊用户会话

### 图片消息

- 钉钉图片消息可进入桌面会话
- Agent 读取的本地图片可回传到钉钉

### 桌面主动发钉钉

在桌面端聊天输入框工具栏点击钉钉图标，可选择成员发送消息；对已识别的群聊，也可按当前实现作为目标使用。

当前行为：

1. 首次主动发送成功后，当前桌面会话绑定到该钉钉目标
2. 后续该目标从钉钉回复时，会优先回到当前绑定会话
3. 若当前会话已关闭，则进入历史选择 / 新建逻辑

## 命令系统

钉钉发送以 `/` 开头的消息，会走命令链路。

### 当前可用命令

| 命令 | 说明 |
|------|------|
| `/help` | 显示帮助 |
| `/status` | 查看当前聊天上下文下的历史会话状态 |
| `/sessions` | 列出活跃会话 |
| `/new [目录]` | 新建会话 |
| `/resume [编号]` | 恢复历史会话 |
| `/rename <名称>` | 重命名当前会话 |
| `/close` | 关闭当前会话 |

### `/status` 的状态标记

- `✅` 当前映射会话
- `🔵` 已激活但不是当前映射
- `⭕` 历史存在但当前未激活

### `/resume` 的行为

1. 不带编号时，返回历史列表
2. 带编号时，直接恢复目标会话
3. 若目标会话已在运行但正在回复，提示“当前正在回复，请等待完成”
4. 若目标会话未激活，则先恢复，再在需要时自动 hello 激活

### `/close` 的行为

- 当前只支持 `/close`
- 不支持 `/close 1` 这类带编号形式

## 会话管理

### 关闭后再次入站

当前桌面会话关闭后，再从同一钉钉聊天发消息：

1. 如果仍存在允许复用的当前绑定，则回到该绑定会话
2. 否则进入历史选择 / 新建逻辑

### 群聊名称

钉钉群聊标题与绑定显示，当前优先使用群名；拿不到群名时回退到群 ID 或本地已知名称，不再默认叠加用户名。

## 桌面端介入

钉钉发起的会话在桌面端可见。桌面端继续发言时：

1. 回复会回传到当前绑定钉钉目标
2. 不会误回到其他渠道

## 常见问题

### 主动发送报 `MissinguserIds`

通常说明当前目标被当成单聊发送，但平台要求提供 `userIds`。应检查：

- 当前目标是否真的是群聊
- `robotCode` 是否已配置
- 群 / 单聊身份是否已经正确建立

### 桌面端发送列表为空

检查：

- `robotCode` 是否已填写
- 通讯录权限是否已开通
- 应用是否已重新发布

### 收到重复回复

检查是否多台桌面端共用同一个机器人。
