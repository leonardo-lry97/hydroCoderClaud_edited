# 钉钉机器人桥接使用指南

通过钉钉机器人桥接功能，你可以在钉钉中直接与 Hydro Desktop 的 Agent 对话，实现远程访问本地 Agent 会话。

## 前提条件

- Hydro Desktop 已安装并正常运行
- Claude Code CLI 已安装且 API 配置正常
- 拥有钉钉组织的管理员权限（用于创建机器人应用）

## 重要限制

> **一个机器人只能绑定一个桌面端。** 同一组织内，同一个机器人的 AppKey/AppSecret 只能在一台 Hydro Desktop 上配置使用。多台桌面端同时连接同一个机器人会导致消息丢失或重复回复。如需多人使用，请为每人创建独立的机器人应用。

## 第一步：创建钉钉机器人应用

1. 登录 [钉钉开放平台](https://open-dev.dingtalk.com/)
2. 进入「应用开发」→「企业内部开发」→「创建应用」
3. 填写应用名称（如 "Hydro Desktop AI"）和描述
4. 创建完成后，在应用详情页获取 **AppKey** 和 **AppSecret**

## 第二步：配置机器人能力

1. 在应用详情页，进入「添加应用能力」→ 添加「机器人」
2. 配置机器人信息：
   - 机器人名称：自定义（如 "Hydro Desktop Agent"）
   - 消息接收模式：选择 **Stream 模式**（非 HTTP 模式）
3. 在「权限管理」中确保已开通以下权限：
   - **上传媒体文件**（用于图片转发）
   - **机器人发消息**（用于回复消息与主动发送）
   - **`qyapi_get_department_list`**（用于桌面端主动发送时拉取组织部门）
   - **`qyapi_get_department_member`**（用于桌面端主动发送时拉取组织成员）
4. 发布应用（未发布的应用无法使用）

### 主动发送最小权限说明

如果你希望从桌面端直接给钉钉组织成员发消息，而不是只能等钉钉侧先发起，那么除了 `AppKey / AppSecret` 外，还必须满足以下条件：

1. 应用权限已开通：
   - `qyapi_get_department_list`
   - `qyapi_get_department_member`
   - 上传媒体文件
   - 机器人发消息
2. 机器人能力已启用
3. 已发布到企业生效
4. 已在 Hydro Desktop 中填写 `robotCode`

说明：

- 缺少 `qyapi_get_department_list` 时，桌面端无法加载组织部门。
- 缺少 `qyapi_get_department_member` 时，桌面端无法加载组织成员。
- 修改权限后，通常需要重新发布应用，桌面端重新打开发送面板后才会体现最新权限。

## 第三步：在 Hydro Desktop 中配置

1. 打开 Hydro Desktop
2. 点击左下角齿轮图标，选择「钉钉设置」
3. 填写配置：

   | 字段 | 说明 |
   |------|------|
   | **启用桥接** | 打开开关 |
   | **AppKey (ClientID)** | 填入第一步获取的 AppKey。钉钉**应用**的凭证，用于鉴权和建立连接 |
   | **AppSecret (ClientSecret)** | 填入第一步获取的 AppSecret，用于配合 AppKey 获取 access_token |
   | **机器人 Code (robotCode)** | 填入钉钉**机器人**的编码。在钉钉开放平台 → 应用能力 → 机器人配置页获取。**注意：robotCode 和 AppKey 是不同的标识**——AppKey 是应用级凭证，robotCode 是机器人级编码，桌面主动发送钉钉消息时必须填写 |
   | **默认工作目录** | Agent 执行命令的工作目录（可选，强烈建议设置，方便查看 Agent 生成的文件和成果） |
   | **历史会话数量** | 钉钉用户选择历史会话时显示的最大数量（默认 5，范围 1-20，在「高级设置」中配置） |

4. 点击「保存」，然后点击「连接」
5. 状态显示「已连接」即配置成功

## 使用方式

### 基本对话

在钉钉中找到机器人，直接发送文本消息即可。机器人会将消息转发给 Agent，并将回复发回钉钉。

### 发送图片给 Agent

在钉钉对话中直接发送图片，Agent 会自动识别图片内容并回复。

### Agent 图片转发

当 Agent 在处理过程中使用工具读取了本地图片文件（如通过 Read 工具），该图片会自动转发到钉钉。

**提示**：如果希望 Agent 生成图片后自动发送到钉钉，可以在提示词中加上"生成后请读取该图片"，确保 Agent 使用 Read 工具读取图片文件。

### 桌面主动发钉钉

在桌面端聊天输入框工具栏点击钉钉图标，可直接选择组织成员发送消息。

当前行为是：

- 首次从某个桌面会话主动发给钉钉成员后，该成员会绑定到当前桌面会话
- 后续该成员从钉钉回复时，会优先回到当前桌面会话
- 如果当前会话已关闭，再回复时才会回到历史/新建逻辑

## 会话管理

### 会话隔离

不同群或单聊发起的对话相互隔离，各自维护独立的 Agent 会话和上下文。同一个群里的消息共用一个会话。

### 历史会话选择

当当前钉钉对话没有绑定活跃会话、但存在历史会话时，机器人会列出历史会话供选择：

```
您有以下历史会话，请回复数字选择：

1. [2小时前] 代码审查任务
2. [昨天] Python 脚本生成

回复 0 开始全新会话
```

- 回复 **0**：创建全新会话，从头开始
- 回复 **1~N**：恢复对应历史会话（包含历史上下文，Agent 可继续之前的对话）

**等待选择期间继续发消息**：机器人会重复显示选择菜单，并以你**最后一条**消息作为选定会话后的第一个问题（前几条消息会被替换）。

### 正在处理时发消息

如果 Hydro Desktop 桌面端正在处理某个请求（Agent 生成回复中），此时钉钉发来的消息会收到提示：

```
⏳ 正在处理中，请稍候再试
```

等 Agent 回复完成后再发送即可。

### 为什么桌面端看不到可发送成员

如果桌面端“发送钉钉消息”面板中没有成员列表，优先检查：

1. 是否已开通 `qyapi_get_department_list`
2. 是否已开通 `qyapi_get_department_member`
3. 权限变更后是否已重新发布应用
4. `robotCode` 是否已正确填写

当前版本会直接显示真实错误原因，不再把接口错误伪装成“空列表”。

## 命令层

在钉钉中发送以 `/` 开头的消息，会被识别为命令，不进入 AI 对话。

### 命令列表

| 命令 | 说明 |
|------|------|
| `/help` | 显示所有可用命令 |
| `/status` | 查看系统状态（连接状态、当前配置、会话数量） |
| `/sessions` | 列出当前所有活跃会话 |
| `/new [目录]` | 新建会话，可选指定工作目录名或绝对路径 |
| `/resume [编号]` | 恢复历史会话，不带编号时显示选择列表 |
| `/rename <名称>` | 修改当前会话名称 |
| `/close` | 关闭当前会话 |

### 使用示例

**新建会话并指定工作目录**

```
/new my-project
```

会在默认输出目录下创建 `my-project` 子目录作为工作目录。

```
/new /Users/yourname/projects/webapp
```

macOS 使用绝对路径直接指定工作目录。

```
/new D:\projects\webapp
```

Windows 使用绝对路径直接指定工作目录。

**恢复历史会话**

```
/resume
```

显示历史会话列表，回复对应编号即可恢复。

```
/resume 2
```

直接恢复列表中第 2 个历史会话。

**重命名当前会话**

```
/rename 前端重构任务
```

**查看系统状态**

```
/status
```

返回示例：

```
📊 系统状态

├─ 钉钉桥接: ✅ 已连接

├─ 当前配置: 官方 API

├─ 执行中: 0 个 / 空闲: 1 个

└─ 总会话数: 1 个
```

### 注意事项

- 命令不区分大小写，`/Help` 和 `/help` 效果相同
- `/resume` 仅在当前会话正在处理中（streaming）时不可用；空闲会话下可直接恢复历史会话
- `/resume` 切换的是当前钉钉对话绑定的会话，不会自动关闭其他空闲会话
- `/rename` 和 `/close` 需要有活跃会话才能执行

## Hydro Desktop 桌面端介入

### 什么是介入

钉钉发起的会话同时在 Hydro Desktop 桌面端可见。当桌面端用户（而非钉钉用户）在该会话窗口中直接输入问题并获得 Agent 回复时，这次交互称为"桌面端介入"。

### 介入内容自动同步

桌面端介入的完整 Q&A 会自动发送到钉钉，格式如下：

```
💻 桌面端介入：
> 帮我检查一下最新的日志文件

日志中发现 3 处异常，分别位于...（Agent 完整回复）
```

如果 Agent 在介入过程中读取了图片文件，图片同样会转发到钉钉。

### 介入的适用场景

- 钉钉用户提了一个需要操作本地文件的复杂问题，Hydro Desktop 桌面端用户直接在桌面端完成并让钉钉用户看到结果
- 钉钉用户不在线时进行的操作，等对方回来时可以在钉钉查看历史记录
- 多人协作：一人发起任务，另一人在桌面端处理，钉钉群自动记录过程

## 常见问题

### 连接失败

- 确认 AppKey 和 AppSecret 是否正确
- 确认应用已发布
- 确认机器人消息接收模式为 Stream 模式
- 检查网络是否能访问钉钉服务器

### 消息无回复

- 确认 Hydro Desktop 状态栏显示「已连接」
- 确认 Claude Code CLI 的 API 配置正常
- 查看 Hydro Desktop 开发者工具（F12）中的控制台日志

### 图片无法转发到钉钉

- 确认已开通「上传媒体文件」权限
- 确认图片文件大小不超过 20MB
- 确认 Agent 确实使用了 Read 工具读取图片（仅文本中提到路径不会触发转发）

### 收到重复回复

- 检查是否有多台桌面端配置了同一个机器人，请确保一个机器人只绑定一个桌面端
