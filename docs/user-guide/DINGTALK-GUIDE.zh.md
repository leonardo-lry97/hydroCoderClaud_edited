# 钉钉桥接配置指南

这份文档只解决一件事：**让用户按步骤把钉钉和 Hydro Desktop 配通**。

## 先准备好这些条件

开始前请先确认：

- 你所在的企业已经开通钉钉
- 你有 **钉钉组织管理员**，或至少有权限创建和发布企业内部应用的账号
- 电脑上已经安装并能正常打开 Hydro Desktop
- Hydro Desktop 里的 Agent / 模型功能已经可正常使用
- 不要让多台桌面端同时连接同一个钉钉机器人

如果你还需要从桌面端主动给钉钉成员发消息，请额外确认：

- 你准备使用的是 **企业内部应用**
- 应用具备机器人能力
- 你可以申请通讯录相关权限

## 第 1 步：在钉钉开放平台创建应用

1. 登录钉钉开放平台
   - 申请入口：https://open-dev.dingtalk.com/
2. 创建 **企业内部开发应用**
3. 给应用添加 **机器人** 能力
4. 创建完成后，记录：
   - `AppKey`
   - `AppSecret`
   - `robotCode`

说明：

- 其中 `robotCode` 对桌面端主动发送很重要
- 如果只配置了 `AppKey` 和 `AppSecret`，但没有 `robotCode`，通常无法正常使用主动发送

## 第 2 步：配置权限

如果只想先跑通基础聊天，先把机器人基础收发能力配好。

如果还要支持桌面端主动给成员发消息，建议至少开通这些方向的权限：

- 上传媒体文件
- 机器人发消息
- 部门列表读取
- 部门成员读取

说明：

- 后台权限名称可能会随着钉钉控制台版本变化略有不同，但用途大致一致
- 权限修改后，通常要重新发布应用

## 第 3 步：发布应用

前面的能力和权限配好后：

1. 在钉钉开放平台提交发布
2. 等待应用生效

注意：

- 如果后面你补开了权限，也通常需要重新发布一次

## 第 4 步：在 Hydro Desktop 中填写配置

打开 Hydro Desktop，进入 **钉钉桥接设置**，按下面填写：

| 字段 | 填什么 |
|------|------|
| 启用钉钉桥接 | 打开 |
| AppKey | 钉钉开放平台里的 `AppKey` |
| AppSecret | 钉钉开放平台里的 `AppSecret` |
| robotCode | 钉钉开放平台里的 `robotCode` |
| 默认工作目录 | 可选，不填也可以 |
| 历史会话数量 | 可选，默认通常就够用 |

填写完成后：

1. 点击 **保存**
2. 点击 **连接**
3. 等待状态变成 **已连接**

## 第 5 步：验证是否配置成功

建议按下面顺序验证：

### 验证 1：单聊

1. 用钉钉测试账号给机器人发一条普通文字消息
2. 看 Hydro Desktop 是否收到消息并创建 / 命中会话
3. 看机器人是否能正常回复

### 验证 2：群聊

1. 把机器人加入测试群
2. 在群里发送消息
3. 确认桌面端能收到，并且群聊消息不会错落到某个单聊用户会话里

### 验证 3：桌面主动发送

1. 在 Hydro Desktop 会话工具栏里点击钉钉发送入口
2. 选择成员发送一条测试消息
3. 再从钉钉回一条消息，确认它能回到原会话

如果这一步没有成员可选，通常是通讯录权限或 `robotCode` 没配对。

## 常见问题

### 明明填了 AppKey 和 AppSecret，为什么还是不对

优先检查：

- 应用是不是企业内部应用
- 机器人能力是否已经开启
- 应用是否已经发布生效
- `robotCode` 是否已经填写

### 为什么桌面端发送列表为空

通常先检查：

- `robotCode` 是否已填写
- 通讯录权限是否已开通
- 权限改完后是否重新发布了应用

### 主动发送时报 `MissinguserIds`

通常说明当前目标被按单聊发送，但平台要求成员 ID。优先检查：

- 当前目标到底是不是群聊
- 群 / 单聊身份是否已经建立正确
- `robotCode` 是否配置正确

### 为什么建议只让一台桌面端连接这个机器人

因为多台桌面端同时连接同一个钉钉机器人时，最容易出现重复回复、抢消息或回复错乱。
