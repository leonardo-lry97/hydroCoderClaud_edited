微信通知配置指南

通过微信 iLink 通道实现桌面端与微信用户的双向消息互通

核心边界

扫码授权目标用户扫码后成为可发送目标
不读通讯录不能给任意好友发消息
文本 + 图片语音、视频、文件暂未实现
长轮询模式时延比钉钉 / 飞书略高
运行状态说明

设置页顶部状态标签表示的是桌面侧微信 bridge 是否在运行，不等于“微信用户是否已经扫码授权”。

1扫码授权与捕获

  1. 打开 Hydro Desktop，进入「能力管理」→「微信通知」
  2. 确认「启用微信通知」处于打开状态
  3. 如有需要，设置「轮询间隔 (ms)」和「轮询超时 (ms)」并保存
  4. 点击「生成授权二维码」，系统显示二维码
  5. 让需要接收消息的微信用户自己扫码并确认授权
  6. 扫码完成后，让该用户再发送一条微信消息
  7. 系统自动捕获目标，该用户出现在可发送列表中，并收到一条绑定成功提示
关键步骤

扫码后必须让微信用户发一条消息，系统才能拿到 contextToken 完成捕获。只扫码不发消息，目标不会出现。

关闭开关会发生什么？

关闭后会停止后台轮询，并隐藏聊天工具栏里的微信入口；但已授权账号和已捕获目标仍会保留，重新开启后可继续使用。

2发送微信消息

三种发送入口：

入口操作方式
能力管理页在「微信通知」页面选择目标，填写内容后发送
Agent 会话工具栏聊天输入框工具栏点击微信图标，选择目标后发送
Notebook 对话工具栏Notebook 对话面板同样支持微信快捷发送

从桌面会话主动发给微信后，该目标会绑定到当前会话，后续回信优先回该会话。

3微信回信路由

  1. 微信用户回复消息
  2. 系统检查该目标是否已绑定桌面会话：
    • 已绑定：回信进入该桌面会话
    • 未绑定：创建或恢复一个「微信」来源的会话
  3. 桌面端收到消息（带「来自微信」标记）
  4. Agent 自动回复，回复发回微信

4定时任务配合

定时任务执行会话会注入微信通知 MCP，任务完成后可主动推送结果给已绑定目标。

  • 每天定时生成摘要后发给微信用户
  • 任务失败时发告警通知
  • 长任务结束后发结果通知

常见问题

扫码后为什么还没有目标？

该微信用户还没有发送第一条消息。先让对方发一条文字消息。

为什么能发给一个人，发给另一个人失败？

另一个人不是有效的已捕获目标。让该用户重新扫码并再发一条消息即可恢复。

微信回信会不会总是新建会话？

不会。只要目标之前从某个桌面会话主动发出过，回信会优先回到原会话。

能否发给没扫码的人？

不能。只能发给已扫码授权且已捕获的目标，不读取微信通讯录。

设置页显示“运行中”是不是代表已经扫码成功？

不是。这只表示桌面侧微信 bridge 正在运行；目标是否可发送，还取决于扫码授权和首条消息捕获是否完成。
