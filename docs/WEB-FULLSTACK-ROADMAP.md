# Hydro Web Fullstack Roadmap

## 目标

把当前已经可真实对话的 Web 版本，逐步演进为完整的 Web 端工作台，覆盖在线对话、文档与网页任务、任务执行、配置管理和多用户能力。

## 第一阶段现状

- 已完成独立 Node 服务端
- 已接入 DeepSeek 服务端代理
- 已完成在线会话、普通发送、流式发送
- 已完成 SQLite 会话持久化
- 已支持参考网页 URL 上下文
- 已支持本地文本文件导入为补充上下文

## 第二阶段

### 目标

把“能聊天”升级为“能完成文档/网页/问答任务”。

### 功能清单

- 会话持久化
- 流式响应
- 参考网页抓取
- 文本文件导入
- 系统提示词管理
- 模型切换
- 基础错误提示
- 对话历史恢复

### 继续补齐项

- 文件分片上传
- PDF / Word / Excel 解析
- 网页抽取质量优化
- 长文上下文压缩
- 引用来源展示
- 基础任务模板

## 第三阶段

### 目标

把在线问答版升级为可执行任务的完整 Web 工作台。

### 服务端模块

- 认证与用户系统
- 权限与多租户
- 任务执行队列
- Tool / Skill 执行网关
- 文件存储与对象存储
- 会话与任务数据库
- 审计日志与监控
- 限流与配额控制

### 前端模块

- 工作台主页
- 对话列表与标签
- 文件中心
- 任务中心
- 模型与 Provider 管理
- Prompt / Skill 管理
- MCP / Hooks / Agents 管理
- 团队与权限管理

### 任务执行能力

- 网页检索与摘要
- 文档分析与总结
- 结构化报告生成
- 文件改写与导出
- 多步骤任务编排
- 长任务状态回传
- 失败重试与恢复

### 数据层建议

- 开发期：SQLite
- 正式版：PostgreSQL
- 文件：本地存储或 S3 兼容对象存储

### 部署建议

- 前端：Vite 构建产物
- 后端：独立 Node 服务
- 反向代理：Nginx / Caddy
- 持久化：数据库与对象存储分离

## 推荐演进顺序

1. 完善第二阶段文件解析与网页抽取
2. 补登录鉴权与用户隔离
3. 增加任务执行网关
4. 接入 Tool / Skill 体系
5. 做工作台与管理台
6. 做正式部署与监控
